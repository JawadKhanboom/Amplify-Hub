# Practical Resource Library — editorial review checklist

> **Publication is gated.** A resource appears in the public library and the public
> database read policy **only** when it is `active` **and** `status: 'reviewed'`.
> Everything below is currently a DRAFT: written for AmplifyHub, self-reviewed by the
> authoring model (see `docs/resource-editorial-selfreview.md`), but **not yet approved
> by a human editor** — and therefore not publicly visible.

## How to approve a resource (per row)

1. Serve the site locally (any static server) and open the **Preview** link — this is the
   exact reader experience, with a draft banner.
2. Check: accurate and beginner-safe advice · no dishonesty, pressure after a firm "no",
   or fabricated-experience coaching · industry-neutral · clear grammar · realistic duration.
3. Skim the downloadable files for the same content (`assets/resources/downloads/<id>.*`).
4. If it passes, edit that entry in `assets/resource-catalog.js` and add `status: 'reviewed'`
   (drafts carry no status field — the catalog defaults them to 'draft').
5. After approving any batch, regenerate: `npm run generate:resources` and
   `npm run generate:resource-migration` (in `sales-mindset-app/`), then run `npm run qa:resources`.
